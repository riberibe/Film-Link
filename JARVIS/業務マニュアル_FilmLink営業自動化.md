## このツールが何をするか（1分で理解）

有料老人ホームなどのお問い合わせフォームに、営業メッセージを**自動で入力して送信**するツールです。

```
手作業だと...
  1施設のフォームを開く → 名前・メール・電話・本文を入力 → 送信
  → これを何千件も繰り返す（1件3〜5分 × 100件 = 約5〜8時間）

このツールだと...
  ブラウザを開いて「送信開始」を押すだけ
  → 並列4で動き、1000件を約25分で自動送信
```

## 1. 毎日の使い方

### 基本の流れ（ターミナル不要）

```
① デスクトップの「ツール起動フォルダ」を開く
        ↓
② 「フォーム送信ツール起動.command」をダブルクリック
        ↓
③ ブラウザが自動で開く（http://localhost:3737）
        ↓
④ 「送信」タブを選ぶ
        ↓
⑤ キャンペーンを選んで「本番送信 開始」を押す
        ↓
⑥ ログが流れ始めたら放置（終わったら自動で止まる）
```

**これだけです。** ターミナルもコマンドも不要。

---

### 実行中のログの見方

```
[1/4910] 有限会社ICHIJO → 送信完了（name, email, phone, body）
[2/4910] 株式会社PLIMES → 送信完了（name, email, phone, body）
[3/4910] 社会福祉法人〇〇 → URLなし・スキップ
[4/4910] 株式会社△△ → エラー（タイムアウト）
```

| 表示 | 意味 |
|---|---|
| 送信完了 | 正常に送信できた ✅ |
| URLなし・スキップ | フォームURLが未登録（送信なし） |
| 送信ボタン未発見 | フォームを埋めたが最終送信に失敗（電話番号のみページ等） |
| エラー | サイトが開けなかった等（次回実行時に自動リトライ） |

> **6〜7割送信できていれば正常です。** サイト構造やロボット対策で弾かれるものは一定数あります。

## 3. 起動の仕組み

### Mac起動時に自動でサーバーが立ち上がる

このツールはMacを起動すると**自動でバックグラウンド起動**するように設定されています（LaunchAgent常駐）。

- `.command` ファイルはブラウザを開くだけ（サーバーを起動しているわけではない）
- サーバー自体は起動済みなので、`.command` を押せばすぐ使える
- Mac負荷はほぼゼロ（RAM 30〜50MB程度、待機中はCPU 0%）
- 送信中だけCPUが上がる（Playwright=ブラウザが複数動くため）

## 4. 充電切れ・Mac再起動後の再開

### 再開手順（完全自動）

```
① Macが再起動する
② 30秒ほど待つ（サーバーが自動起動する時間）
③ 「フォーム送信ツール起動.command」をダブルクリック
④ 「送信開始」を押す → 続きから自動再開
```

### 重複送信の仕組み

送信済みの施設は `campaigns/01_kaigo/results.csv` に記録されています。  
「送信開始」を押すたびにこのファイルを読んで、**記録済みのURLは自動スキップ**します。

```
例: 1000件処理済みで充電切れ
→ 再起動後に「送信開始」を押すと 1001件目から開始
```

### バッテリー切れ中に送信されたものの記録が残らなかった場合

`screenshots/` フォルダに `ERR_XXXX_施設名.png` として処理履歴が残っています。  
エラーになった施設だけがキャプチャされるため、「エラーが出ていない＝送信成功」として判断できます。

## 5. 送信設定を変えたいとき

GUIの「設定」タブから変更できます。

| 設定項目 | 意味 | 推奨値 |
|---|---|---|
| **1日の送信上限** | 1回の実行で送る最大件数 | 3000 |
| **並列数** | 同時に開くブラウザタブ数 | 4 |
| **施設間の待機時間** | 1送信ごとに何ミリ秒待つか | 6000（6秒） |
| **送信開始時刻** | この時刻より前は送信しない | 09:00 |
| **送信終了時刻** | この時刻を過ぎたら翌朝まで待機 | 18:00 |

### 並列数と時間の目安

| 並列数 | 1000件にかかる時間 |
|--------|----------------|
| 1 | 約100分 |
| 4（推奨） | 約25分 |
| 8 | 約13分（Bot検知リスク高） |

## 7. 結果の確認方法

### GUIの「結果」タブ

ブラウザから直接確認できます。送信完了 / エラー / スキップ でフィルタ可能。

### results.csv を直接見る

`campaigns/01_kaigo/results.csv` をExcel（またはNumbers）で開いて「送信結果」列を確認。

### エラー時のスクリーンショット

`screenshots/` フォルダ内の `ERR_XXXX_施設名.png` がエラー分。  
ツールが自動でキャプチャを保存しています。

### 自動返信メール

送信成功した施設のうち、自動返信設定がある施設から確認メール（`cv.contact000@gmail.com`）が届きます。  
自動返信がない施設も多いので、メール数＝送信数にはなりません。

## 9. よくあるトラブルQ&A

**Q: 「フォーム送信ツール起動.command」を押してもブラウザが開かない**

サーバーがまだ起動中の可能性があります。30秒ほど待ってから再度ダブルクリックしてください。  
それでもダメな場合はMacを再起動してください。

---

**Q: 送信が途中で止まった（ブラウザを閉じた・ネット切れ等）**

もう一度「送信開始」を押せば続きから再開します。送信済み分は重複しません。

---

**Q: 充電が切れて止まった。最初からやり直さないといけないか**

やり直し不要です。再起動後に「送信開始」を押せば未送信分から自動で続きます。

---

**Q: 同じ施設に2回送ってしまわないか**

送信済みのURLは `results.csv` に記録されており、次回実行時は自動スキップされます。  
電源断・クラッシュで記録が残らなかった場合も、`screenshots/` フォルダが処理履歴として機能します。

---

**Q: 送信完了が少ない（3〜4割しか送れていない）**

6〜7割が正常範囲です。フォームなし（電話番号のみ）のサイトや、ロボット対策が原因です。  
`screenshots/` フォルダのエラーキャプチャを確認して、Jarvisに報告してください。

---

**Q: エラーになった施設はどうなる**

次回「送信開始」を押したとき、エラー施設は `results.csv` に「送信完了」として記録されていないため、自動で再試行されます。
