# FilmLink フォーム自動送信ツール — 完全マニュアル

> 対象: リベ様 / 運用スタッフ（エンジニアじゃなくても使えます）  
> 最終更新: 2026-04-27

---

## このツールが何をするか（1分で理解）

有料老人ホームなどのお問い合わせフォームに、営業メッセージを**自動で入力して送信**するツールです。

```
手作業だと...
  1施設のフォームを開く → 名前・メール・電話・本文を入力 → 送信
  → これを何十件も繰り返す（1件3〜5分 × 50件 = 約3時間）

このツールだと...
  コマンド1行打つだけ → 50件を約10〜15分で自動送信して止まる
```

## 1. ファイル構成

ツールは以下の5つのファイルで動いています。

```
ファミリームービー/ツール/form_sender/
│
├── send_forms.js   ← ツール本体（触らなくてOK）
├── facilities.csv  ← 送信先リスト（施設名・フォームURLなど）
├── config.json     ← 設定ファイル（文面・件数・送信者情報はここを編集）
├── results.csv     ← 送信結果のログ（実行すると自動で作られる）
└── screenshots/    ← 送信直前の画面キャプチャ（自動で作られる）
```

**基本的に操作するのは `config.json` と `facilities.csv` の2つだけです。**

## 2. 初回セットアップ

> この作業は最初の1回だけです。1回やれば次からは不要です。

### Mac の場合

#### ステップ1: ターミナルを開く

ターミナルとは、コマンドを入力する黒い画面のことです。

**開き方:**
1. 画面右上の虫眼鏡アイコン（Spotlight検索）をクリック
2. 「ターミナル」と入力して Enter
3. 黒い画面（ターミナル）が開く

#### ステップ2: Node.js がインストール済みか確認する

ターミナルに以下を入力して Enter を押す。

```
node --version
```

- `v20.x.x` のような数字が出た → **インストール済みなのでステップ3へ**
- `command not found` と出た → **次のステップでインストールが必要**

#### ステップ2a: Node.js をインストールする（未インストールの場合のみ）

1. ブラウザで `nodejs.org` を開く
2. 「LTS」と書いてある緑のボタンをクリックしてダウンロード
3. ダウンロードされた `.pkg` ファイルをダブルクリック
4. 「続ける」を押し続けて「インストール」まで進める
5. 完了後、ターミナルで `node --version` を再実行して数字が出ればOK

#### ステップ3: ツールフォルダに移動する

ターミナルに以下を**そのままコピペ**して Enter。

```
cd /Users/ribetyan/Desktop/ファミリームービー/ツール/form_sender
```

#### ステップ4: 必要なパーツをインストールする

以下を**1行ずつ**入力して Enter。（各コマンドが完了するまで待ってから次へ）

```
npm install
```
（完了まで1〜3分かかります）

```
npx playwright install chromium
```
（完了まで3〜5分かかります。Chromeに似たブラウザをインストールしています）

#### ステップ5: 動作確認

```
node send_forms.js --dry
```

Chromeが自動で立ち上がって施設サイトを開いたら成功です。  
最後に「ドライラン完了: ○件」と表示されればセットアップ完了。

## 3. 毎日の使い方

### 本番送信の手順

**Mac の場合:**

1. ターミナルを開く（Spotlight検索で「ターミナル」）
2. 以下をコピペして Enter

```
cd /Users/ribetyan/Desktop/ファミリームービー/ツール/form_sender && node send_forms.js
```

**Windows の場合:**

1. PowerShell を開く
2. 以下を1行ずつ実行

```
cd C:\FilmLink\form_sender
node send_forms.js
```

**これだけです。** あとはパソコンが自動でやります。

### 実行中の画面の見方

```

FilmLink フォーム送信ツール
モード: 全自動送信
```

| 表示 | 意味 |

### 途中で止めたいとき

ターミナル（またはPowerShell）の画面で **`Ctrl + C`** を押す。  
途中まで送信した分は記録されているので、次回実行時は続きから自動再開します。

## 5. 文面・送信者情報を変えたいとき

`config.json` というファイルを編集します。

### ファイルを開く方法

**Mac:**
1. Finder で `ファミリームービー/ツール/form_sender/` フォルダを開く
2. `config.json` を右クリック → 「このアプリケーションで開く」→「テキストエディット」

**Windows:**
1. エクスプローラーで `C:\FilmLink\form_sender\` を開く
2. `config.json` を右クリック → 「メモ帳で開く」

### 編集できる項目

```json
{
  "sender": {
    "name": "← 送信者の名前",
    "email": "← 送信者のメールアドレス",
    "phone": "← 送信者の電話番号"
  },
  "message": {
    "subject": "← メールの件名（フォームに件名欄がある場合）",
    "body": "← 本文（\nは改行を表す）"
  }
}
```

### 現在の設定値

| 項目 | 現在の値 |
|---|---|
| 送信者名 | 小林 |
| メール | cv.contact000@gmail.com |
| 電話 | 080-4687-3729 |
| 件名 | ご入居者様からご家族への「映像メッセージ」オプション — FilmLinkよりご提案 |

### 注意事項

- 文中で改行したい場所は `\n` と書く（Enterキーではなく、バックスラッシュ＋nの2文字）
- 保存は `Ctrl + S`（Mac は `Command + S`）
- **文面・料金表現の変更はリベ様の確認必須**

## 6. 1日の送信件数を変えたいとき

`config.json` の `daily_limit` の数字を変更します。

```json
"settings": {
  "daily_limit": 50,   ← この数字を変える
  ...
}
```

**件数の目安:**

| 件数 | 所要時間 | 推奨シーン |
|---|---|---|
| 20件 | 約3〜5分 | テスト・様子見のとき |
| 50件 | 約8〜12分 | 通常運用（デフォルト） |
| 100件 | 約15〜25分 | リスト消化を急ぐとき |
| 300件 | 約1時間 | フル稼働時（反響確認後に変更予定） |

> 300件/日以上を毎日続けるとロボット検知（CAPTCHA）が発動するサイトがあります。エラーとして記録されスキップされるので致命的ではありませんが、送信ペースは状況を見ながら調整してください。

## 7. 結果の確認方法

### 送信ログを見る（results.csv）

1. `form_sender/` フォルダを開く
2. `results.csv` をダブルクリックしてExcel（またはNumbers）で開く
3. 「送信結果」列を確認

### 送信前の画面キャプチャを見る（screenshots フォルダ）

`form_sender/screenshots/` フォルダを開くと、送信直前の画面が全件分保存されています。  
「ちゃんとフォームに入力できていたか」を目視確認できます。

### CRM（スプレッドシート）の確認

送信済み施設のステータスは `facilities.csv` に自動記録されます。  
週1回程度、Jarvisにスプレッドシートへの反映を依頼してください。

## 8. 新しい業種を追加するとき

結婚式場・写真スタジオなど新業種に展開するときの手順です。

### ステップ1: スプレッドシートに新しいシートを追加

1. 営業管理CRMを開く  
   → `https://docs.google.com/spreadsheets/d/19vyVMkO2BsgNLH_ECjwYZ4esbS8Bq63nYYFWhf3oxX4`
2. 画面左下の `+` ボタンをクリックして新シートを追加
3. シート名を業種名にする（例: `営業管理_結婚式場`）
4. 既存シートと同じ列構成でデータを入力（Jarvisが代わりに収集します）

### ステップ2: CSVとしてダウンロード

1. 追加したシートを開いた状態で
2. メニュー「ファイル」→「ダウンロード」→「カンマ区切り形式（.csv）」

### ステップ3: facilities.csv を差し替えて実行

1. ダウンロードしたCSVを `form_sender/` フォルダにコピー
2. 既存の `facilities.csv` を `facilities_老人ホーム.csv` などにリネームしてバックアップ
3. ダウンロードしたファイルを `facilities.csv` に改名
4. `config.json` の文面を業種に合わせて変更（**変更前にリベ様確認**）
5. 実行

## 9. よくあるトラブルQ&A

**Q: 「node は内部コマンドとして認識されません」と出る（Windows）**

Node.js がインストールされていません。セットアップのステップ2aからやり直してください。

---

**Q: 「command not found: node」と出る（Mac）**

Node.js がインストールされていません。セットアップのステップ2aからやり直してください。

---

**Q: Chrome が途中で固まる・タイムアウトになる**

そのサイト側の問題です。ツールはエラーを記録して次の施設に自動で進みます。  
エラーが多発する場合はJarvisに報告してください。

---

**Q: 「CAPTCHA」の画面が出て止まる**

ロボット検知が発動しました。エラーとして記録されスキップされます。  
件数を減らすか、しばらく間隔を空けて再送します。

---

**Q: 途中で止めてしまった。続きから再開できるか**

できます。もう一度 `node send_forms.js` を実行するだけで、  
送信済みの施設は自動でスキップされ、未送信の続きから再開します。

---

**Q: 同じ施設に2回送ってしまった**

`facilities.csv` を開いて、その施設の「ステータス」列を確認してください。  
送信済みは「送信済み」になっているはずです。誤って「未処理」に書き換えると再送されるので注意。
