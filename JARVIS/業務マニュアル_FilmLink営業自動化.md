> 最終更新: 2026-04-27

---

## このツールが何をするか（1分で理解）

有料老人ホームなどのお問い合わせフォームに、営業メッセージを**自動で入力して送信**するツールです。

```
手作業だと...
  1施設のフォームを開く → 名前・メール・電話・本文を入力 → 送信
  → これを何十件も繰り返す（1件3〜5分 × 50件 = 約3時間）

このツールだと...
  コマンド1行打つだけ → 50件を約10〜15分で自動送信して止まる
```

## 1. ファイル構成

ツールは以下のファイルで動いています。

```
ファミリームービー/ツール/form_sender/
│
├── send_forms.js   ← ツール本体（触らなくてOK）
├── facilities.csv  ← 送信先リスト（Jarvisが管理・追加する）
├── config.json     ← 設定ファイル（文面・件数・送信者情報）
├── results.csv     ← 送信結果のログ（実行すると自動で作られる）
└── screenshots/    ← エラー発生時のみ画面キャプチャを保存
```

**リベ様が操作するのは基本的にコマンド1行だけです。**  
施設リストの追加・管理はJarvisが担当します。

## 2. 初回セットアップ

> この作業は最初の1回だけです。1回やれば次からは不要です。

### Mac の場合

#### ステップ1: ターミナルを開く

1. 画面右上の虫眼鏡アイコン（Spotlight検索）をクリック
2. 「ターミナル」と入力して Enter
3. 黒い画面（ターミナル）が開く

#### ステップ2: Node.js がインストール済みか確認する

```
node --version
```

- `v20.x.x` のような数字が出た → **インストール済みなのでステップ3へ**
- `command not found` と出た → **次のステップでインストールが必要**

#### ステップ2a: Node.js をインストールする（未インストールの場合のみ）

1. ブラウザで `nodejs.org` を開く
2. 「LTS」と書いてある緑のボタンをクリックしてダウンロード
3. ダウンロードされた `.pkg` ファイルをダブルクリック
4. 「続ける」を押し続けて「インストール」まで進める
5. 完了後、ターミナルで `node --version` を再実行して数字が出ればOK

#### ステップ3: ツールフォルダに移動する

ターミナルに以下を**そのままコピペ**して Enter。

```
cd /Users/ribetyan/Desktop/ファミリームービー/ツール/form_sender
```

#### ステップ4: 必要なパーツをインストールする

```
npm install
```
（完了まで1〜3分かかります）

```
npx playwright install chromium
```
（完了まで3〜5分かかります。Chromeに似たブラウザをインストールしています）

## 3. 毎日の使い方

### 基本の流れ

```
① Jarvisに「次の施設を追加して」と伝える
      ↓
② Jarvisが facilities.csv に施設を追加する
      ↓
③ ターミナルでコマンド1行打つ
      ↓
④ 自動送信スタート（終わったら自動で止まる）
```

### 本番送信コマンド

以下を**そのままコピペ**してターミナルに貼り付けて Enter。

```
cd /Users/ribetyan/Desktop/ファミリームービー/ツール/form_sender && node send_forms.js
```

**これだけです。** あとはパソコンが自動でやります。

---

### 実行中の画面の見方

```
==================================================
FilmLink フォーム送信ツール
モード: 全自動送信
未処理件数: 48件 / 本日の上限: 50件
==================================================

[1/48] チャームプレミア グラン 松濤
  → 送信完了（入力: company, name, email, phone, body）
[2/48] 東急ウェリナケア自由が丘
  → 送信完了（入力: name, phone, email）
[3/48] グッドタイムリビング大阪ベイ
  → 送信ボタン未発見
...

本日の送信上限（50件）に達しました。残り○件は明日以降に送信されます。
```

| 表示 | 意味 |
|---|---|
| 送信完了 | 正常に送信できた ✅ |
| 送信ボタン未発見 | 入力はできたが送信ボタンを押せなかった（少数発生は正常範囲） |
| URLなし・スキップ | フォームURLが未登録（Jarvisが後で対応） |
| エラー | サイトが開けなかった等（次回実行時に自動リトライ） |

> **6〜7割送信できていれば正常です。** サイト構造やロボット対策で弾かれるものは一定数あります。

---

### 途中で止めたいとき

ターミナルの画面で **`Ctrl + C`** を押す。  
途中まで送信した分は記録されているので、次回実行時は続きから自動再開します。

## 4. Jarvisへの施設追加依頼の仕方

### 基本の頼み方

施設のリストをJarvisに渡すか、以下のように伝えるだけでOKです。

```
「次のバッチを追加して。東京都の有料老人ホーム50件」

「このURLの施設リストを追加して → （URLを貼る）」

「スプレッドシートのNo.70〜120を追加して」
```

Jarvisが自動で `facilities.csv` に追加し、「追加完了、実行してください」と伝えます。

### Jarvisが施設を追加するときのルール

- ベネッセ系（アリア・くらら・グランダ）は除外（ロボット対策でフォームが使えない）
- 既に送信済みの施設は追加しない（重複防止）
- チェーンごとの正しいフォームURLを自動でセット

## 5. 文面・送信者情報を変えたいとき

`config.json` というファイルを編集します。

### ファイルを開く方法

Finder で `ファミリームービー/ツール/form_sender/` フォルダを開き、  
`config.json` を右クリック → 「このアプリケーションで開く」→「テキストエディット」

### 編集できる項目

```json
{
  "sender": {
    "name": "← 送信者の名前",
    "email": "← 送信者のメールアドレス",
    "phone": "← 送信者の電話番号"
  },
  "message": {
    "subject": "← メールの件名（フォームに件名欄がある場合）",
    "body": "← 本文（\nは改行を表す）"
  }
}
```

## 6. 1日の送信件数を変えたいとき

`config.json` の `daily_limit` の数字を変更します。

```json
"settings": {
  "daily_limit": 50,   ← この数字を変える
  ...
}
```

**件数の目安:**

| 件数 | 所要時間 | 推奨シーン |
|---|---|---|
| 50件 | 約10〜15分 | 通常運用（デフォルト） |
| 100件 | 約20〜30分 | リスト消化を急ぐとき |
| 300件 | 約1〜1.5時間 | フル稼働時（反響確認後） |

> 300件/日以上を毎日続けるとロボット検知（CAPTCHA）が発動するサイトがあります。エラーとして記録されスキップされるので致命的ではありませんが、送信ペースは状況を見ながら調整してください。

## 7. 結果の確認方法

### 送信ログを見る（results.csv）

1. `form_sender/` フォルダを開く
2. `results.csv` をダブルクリックしてExcel（またはNumbers）で開く
3. 「送信結果」列を確認

### エラー時のスクリーンショット（screenshots フォルダ）

エラーが起きた施設のみ、送信直前の画面キャプチャが保存されます。  
ファイル名が `ERR_` から始まるものがエラー分です。容量は最小限に抑えています。

### CRM（スプレッドシート）への反映

`facilities.csv` 内のステータスは自動更新されます。  
Googleスプレッドシート側への反映はJarvisに依頼してください。

## 8. 新しい業種を追加するとき

結婚式場・写真スタジオなど新業種に展開するときは、Jarvisに以下のように伝えます。

```
「次は結婚式場に展開したい。リストを作って送信設定もしてほしい」
```

Jarvisが：
1. 対象施設のリストを収集
2. フォームURLを調査・設定
3. 文面の変更提案
4. facilities.csv に追加

まで対応します。

## 9. よくあるトラブルQ&A

**Q: 「command not found: node」と出る（Mac）**

Node.js がインストールされていません。セットアップのステップ2aからやり直してください。

---

**Q: Chrome が途中で固まる・タイムアウトになる**

そのサイト側の問題です。ツールはエラーを記録して次の施設に自動で進みます。  
エラーが多発する場合はJarvisに報告してください。

---

**Q: 「CAPTCHA」の画面が出て止まる**

ロボット検知が発動しました。エラーとして記録されスキップされます。  
件数を減らすか、しばらく間隔を空けて再送します。

---

**Q: 途中で止めてしまった。続きから再開できるか**

できます。もう一度 `node send_forms.js` を実行するだけで、  
送信済みの施設は自動でスキップされ、未送信の続きから再開します。

---

**Q: 送信完了が少ない（3〜4割しか送れていない）**

6〜7割が正常範囲です。サイト構造・ロボット対策・フォーム形式の違いによるものなので、  
それ以下が続く場合はJarvisに結果ファイルを共有して報告してください。

---

**Q: 同じ施設に2回送ってしまわないか**

送信済みの施設は `facilities.csv` の「ステータス」が「送信済み」に自動更新されます。  
次回実行時は自動スキップされるので二重送信は起きません。
