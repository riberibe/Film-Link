# FilmLink スクレイパー（リストビルダー）— 完全マニュアル

> 対象: リベ様 / 運用スタッフ（プログラムを知らなくても使えます）  
> 最終更新: 2026-04-28

---

## このツールが何をするか（1分で理解）

営業先企業のリストを**自動で収集するツール**です。

```
手作業だと...
  「有料老人ホーム 東京」で検索 → 1件ずつサイトを開く
  → 社名・住所・URLを手でメモ → お問い合わせページを探す
  → これを100件繰り返す（数時間かかる）

このツールだと...
  コマンド1行打つだけ → 100件のリストを自動収集
  → CSVとGoogleスプレッドシートに自動保存
  → そのまま form_sender（自動送信ツール）に読み込ませて送信できる
```

## 1. ファイル構成

ツールは以下のファイルで動いています。

```
ファミリームービー/ツール/form_sender/
│
├── scraper/                     ← スクレイパーツール本体のフォルダ
│   ├── index.js                 ← 【入口】コマンドを受け取って全体を動かす
│   ├── orchestrator.js          ← 【管理】複数ブラウザを並列で動かす司令塔
│   ├── chain_patterns.js        ← 【知識】チェーン企業ごとのフォームURL一覧
│   ├── form_resolver.js         ← 【判定】フォームURLをどこにするか決める部品
│   ├── csv_manager.js           ← 【保存】結果をCSVファイルに書き込む部品
│   ├── sheets_manager.js        ← 【連携】GoogleスプレッドシートにリアルタイムでUIを書き込む部品
│   └── adapters/
│       └── lifull_kaigo.js      ← 【現場】LIFULL介護サイトを実際に操作する部品
│
├── facilities_kaigo.csv         ← 老人ホーム収集結果（ツールが自動生成）
├── facilities_wedding.csv       ← 結婚式場収集結果（業種追加時に自動生成）
├── config.json                  ← 全体の設定ファイル（Google認証情報なども含む）
└── send_forms.js                ← 営業送信ツール（別のツール）
```

**リベ様が操作するのはコマンド1行だけです。ファイルを直接触る必要はありません。**

## 2. 各ファイルの役割

### 「エントリーポイント（entry point）」って何？

**エントリーポイント**とは、日本語で「**入口**」のことです。

プログラムはたくさんの部品（ファイル）に分かれています。  
コマンドを打ったとき、**最初に読み込まれるファイル**のことをエントリーポイントと呼びます。

このツールのエントリーポイントは **`index.js`** です。  
「老人ホームを東京都で100件」という指示が `index.js` に届き、  
そこから各部品に「じゃあ君はこれをやって」と仕事が振られていきます。

---

### ファイルを上から順番に解説（プログラムなしで理解できる版）

---

#### 📌 index.js（入口・受付）

**役割:** あなたが打ったコマンドを最初に受け取る「受付窓口」です。

コマンドに書かれた内容（業種・都道府県・件数・並列数）を読み取って、  
「LIFULL介護アダプターを使って、東京都で、100件、4並列でお願い」と  
後ろの部品に指示を出します。

```
あなたが打つコマンド
    ↓
  index.js が受け取る（ここがエントリーポイント）
    ↓
  orchestrator.js に仕事を渡す
```

---

#### 🎛️ orchestrator.js（管理職・指揮者）

**役割:** 複数のブラウザを同時に動かす「管理職」です。

1件ずつ順番に処理すると時間がかかるため、  
ブラウザを4つ同時に起動して並列で作業させます。  
進捗の表示（「20/100件完了」など）もここが担当します。

```
100件の施設リスト
    ↓
  ブラウザ1 → 25件担当
  ブラウザ2 → 25件担当  ← 4つ同時に動く
  ブラウザ3 → 25件担当
  ブラウザ4 → 25件担当
```

---

#### 🏗️ adapters/lifull_kaigo.js（現場担当・LIFULL介護専用）

**役割:** LIFULL介護のサイトを実際に操作する「現場担当」です。

人間がブラウザでやるのと同じ操作（ページを開く・スクロール・次ページへ進む）を  
自動でやってくれます。施設名・住所・月額費用・公式サイトURLを取得します。

業種ごとにこのファイルが1つあります。  
「工務店用」「結婚式場用」などは今後ここに追加していきます。

---

#### 🔍 form_resolver.js（フォームURL判定）

**役割:** 集めた施設情報をもとに「どのフォームに送るか」を決める部品です。

施設名やURLを見て、`chain_patterns.js` に書かれたチェーンパターンと照合します。

```
例）施設名に「ベネッセ」が含まれる → 手動対応フラグを立てる
    施設名に「ツクイ」が含まれる   → ツクイ専用フォームURLをセット
    どれにも当てはまらない         → 公式サイトURLをそのままセット
```

---

#### 📋 chain_patterns.js（チェーン知識リスト）

**役割:** 「このチェーンのフォームはここ」という知識を持つ設定ファイルです。

エンジニアでなくても中を見れば内容が分かります。  
新しいチェーンを発見したら Jarvis に伝えれば追加できます。

---

#### 💾 csv_manager.js（CSV担当）

**役割:** 収集した施設情報をCSVファイルに書き込む部品です。

同じ施設が重複しないようにチェックしながら、  
通し番号を振って `facilities_kaigo.csv` などに1件ずつ追記します。  
途中でツールが止まっても、それまでに書き込んだ分は失われません。

---

#### 📊 sheets_manager.js（Googleスプレッドシート担当）

**役割:** CSVに書き込むのと同時に、Googleスプレッドシートにも自動で書き込む部品です。

業種ごとにシート（タブ）を自動作成して、  
1件追加されるたびにリアルタイムでスプレッドシートに反映します。  
スプレッドシートを開きながら実行すると、リストが増えていくのをリアルタイムで見られます。

## 3. ツール全体の流れ

コマンドを1行打つと、以下の2フェーズが自動で実行されます。

```
【フェーズ1】施設リストを収集する（単一ブラウザ・順次）

    LIFULL介護のページを順番に開いて
    施設名・都道府県・市区町村・月額費用・施設詳細ページURLを取得
        ↓
    「100件のリストができました」

【フェーズ2】詳細情報を取得 + 書き込み（4ブラウザ並列）

    4つのブラウザが同時に動いて
    各施設の公式サイトURLを詳細ページから取得
        ↓
    フォームURLを判定
        ↓
    CSV + Googleスプレッドシートに追記
        ↓
    進捗表示「50/100件完了」
```

## 5. 基本的な使い方

### コマンドの構成と各パーツの意味

```
node scraper/index.js --industry 老人ホーム --pref 東京都 --limit 100 --workers 4
```

コマンドは5つのパーツで構成されています。順番に解説します。

| # | パーツ | 書き方の例 | 意味 |
|---|---|---|---|
| 1 | `node scraper/index.js` | （固定・変えない） | プログラムを起動するための呪文。毎回このまま使う |
| 2 | `--industry` | `老人ホーム` | 収集する業種を指定する |
| 3 | `--pref` | `東京都` | 対象の都道府県を指定する。**省略すると全国が対象** |
| 4 | `--limit` | `100` | 最大何件収集するか。**省略すると100件** |
| 5 | `--workers` | `4` | 同時に動かすブラウザの数。**省略すると4** |

> 「`--`（ダブルハイフン）」はオプションを指定するときの記号です。  
> `--industry 老人ホーム` は「industryというオプションに老人ホームを指定する」という意味です。

---

### 実行例

```bash
# 東京都の老人ホームを100件
node scraper/index.js --industry 老人ホーム --pref 東京都 --limit 100

# 全国の老人ホームを500件（都道府県省略＝全国）
node scraper/index.js --industry 老人ホーム --limit 500

# 大阪府の老人ホームを200件・ブラウザ6本で高速収集
node scraper/index.js --industry 老人ホーム --pref 大阪府 --limit 200 --workers 6

# 少量テスト（5件だけ試す）
node scraper/index.js --industry 老人ホーム --pref 東京都 --limit 5
```

```

============================================================
```

| 表示 | 意味 |

| スキップ | 前回すでに取得済みだった（重複）件数 |
| エラー | サイトが開けなかった等で取得できなかった件数 |

### 途中で止めたいとき

ターミナルで **`Ctrl + C`** を押す。  
それまでに保存された分はCSVとスプレッドシートに残ります。  
次回実行時は重複チェックにより続きから収集できます。

## 9. よくあるトラブルQ&A

**Q: 「エラー: 業種 "○○" のアダプターはまだ未実装です」と出る**

まだ対応していない業種です。Jarvisに「○○のアダプターを追加してほしい」と依頼してください。

---

**Q: 施設が0件で終わってしまった**

LIFULL介護のサイト構造が変わった可能性があります。  
`scraper_debug_pref13_p1.png` というファイルが自動保存されているので、  
Jarvisに「デバッグ画像を確認して直してほしい」と伝えてください。

---

**Q: エラーが多い（10件以上）**

インターネット回線の問題か、サイト側のアクセス制限の可能性があります。  
`--workers 2` に減らして再実行してください:

```bash
node scraper/index.js --industry 老人ホーム --pref 東京都 --limit 100 --workers 2
```

---

**Q: 途中で止まってしまった。最初からやり直し？**

不要です。重複チェック機能があるため、同じコマンドを再実行すると  
すでに収集済みの施設はスキップして続きから収集します。

---

**Q: スプレッドシートに反映されない**

Google認証のトークンが期限切れの可能性があります。  
Jarvisに「スプレッドシート連携を再認証してほしい」と伝えてください。

---

**Q: 出力されたCSVをExcelで開いたら文字化けした**

1. Excelを開く
2. 「データ」タブ → 「テキストまたはCSVから」
3. ファイルを選択
4. 文字コード「UTF-8」を選んでインポート
