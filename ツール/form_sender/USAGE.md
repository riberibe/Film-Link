# FilmLink フォーム一斉送信ツール — 全機能ガイド

---

## 全体像

SHODANなどで取得した施設CSVを読み込み、各施設のお問い合わせフォームへ自動で文面を送信するツール。

**基本フロー:**
```
SHODANでCSV取得
  → URL生存確認（check_alive_all.js）→ alive.csv 生成
    → ドライランで文面確認（send_forms.js --dry）
      → 本番送信（send_forms.js）
```

## 起動方法

### GUIツール（推奨）

**Mac起動時に自動でサーバーが起動**する（LaunchAgent常駐）。

デスクトップの「ツール起動フォルダ」内の **`フォーム送信ツール起動.command`** をダブルクリック。  
ブラウザが自動で開き `http://localhost:3737` にアクセスできる。

> .commandはブラウザを開くだけ。サーバー本体はMac起動時から常時稼働中。

ログファイルの場所:
```
~/Desktop/ツール起動フォルダ/サーバーログ.log
~/Desktop/ツール起動フォルダ/サーバーエラーログ.log
```

### ターミナルから直接実行

```bash
cd /Users/ribetyan/Desktop/ファミリームービー/ツール/form_sender

# URL生存確認（SHODANのCSVから alive.csv を生成）
node check_alive_all.js campaigns/01_kaigo/<SHODANのCSVファイル名>.csv

# ドライラン 3件（入力まで・送信しない）
node send_forms.js --campaign campaigns/01_kaigo --sample 3 --dry

# サンプル送信 3件（実際に送信）
node send_forms.js --campaign campaigns/01_kaigo --sample 3

# 全件送信
node send_forms.js --campaign campaigns/01_kaigo
```

|------|-----------|

## 設定項目（config.json）

| 項目 | 意味 | 推奨値 |
|------|------|--------|
| `sender.name` | フォームの「氏名」欄に入る名前 | 田中　惇谷 |
| `sender.last_name` | 姓/名が分かれているフォームの「姓」欄 | 田中 |
| `sender.first_name` | 姓/名が分かれているフォームの「名」欄 | 惇谷 |
| `sender.last_name_kana` | カナ分割フォームの「セイ」欄 | タナカ |
| `sender.first_name_kana` | カナ分割フォームの「メイ」欄 | アツヤ |
| `sender.email` | 返信先メール | — |
| `sender.phone` | 電話番号欄 | — |
| `sender.fax` | FAX欄（電話番号と同じでOK） | — |
| `sender.postal_code` | 郵便番号欄 | 262-0032 |
| `sender.address` | 住所欄 | 千葉県千葉市… |
| `sender.company` | 会社名欄 | FilmLink |
| `settings.daily_limit` | 1日の送信上限件数 | 3000 |
| `settings.parallel_contexts` | 同時に処理するタブ数（並列数） | 4 |
| `settings.delay_between_sites_ms` | 1送信ごとの待機時間（ミリ秒） | 6000 |
| `settings.send_window.start` | 送信を開始する時刻 | 09:00 |
| `settings.send_window.end` | 送信を停止する時刻 | 18:00 |

### 並列数と待機時間の関係

| 並列数 | 待機時間 | 1000件の目安時間 | リスク |
|--------|---------|----------------|--------|
| 1 | 6000ms | 約100分 | 低 |
| 4 | 6000ms | 約25分 | 中 |
| 8 | 6000ms | 約13分 | 高（Bot検知されやすい） |

**推奨: 並列4 / 待機6000ms** でスタートし、エラー率が高ければ並列を下げる。

## GUIに出ていない機能（コードレベル）

### send_forms.js の自動処理

| 機能 | 内容 |
|------|------|
| **フォームURL自動探索** | 直接フォームがなくても、サイト内の「お問い合わせ」リンクを自動で辿る |
| **中間ページ突破** | 「お問合せフォームへ」「法人」「一般」「その他」のような分岐ページを自動クリック |
| **2段階フォーム対応** | 「確認画面へ」→「送信する」の2ステップフォームを自動で踏む |
| **ドロップダウン自動選択** | 問い合わせ種別のセレクトボックスは「その他」「ご相談」を優先選択 |
| **姓/名の自動分割入力** | 統合フォーム・分割フォームを自動判別して適切に入力 |
| **カナ対応** | セイ/メイが分かれているフォームにもカナを自動入力 |
| **プライバシー同意チェック** | 「プライバシー」「利用規約」「同意」「承諾」「accept」などのチェックボックスを自動チェック |
| **送信済みスキップ** | results.csv に記録されたURLは次回実行時に自動でスキップ（重複送信なし） |
| **エラー時スクリーンショット** | 送信失敗した施設のページをPNG保存（`./screenshots/` に自動生成） |
| **途中停止対応** | 途中でCtrl+Cしてもそこまでの results.csv は保存済み |
| **多段フォーム対応** | セレクト選択後に入力欄が動的に出現するタイプにも対応 |
| **時間帯制限** | 設定した時間帯（09:00〜18:00）以外は自動で待機、時間になったら再開 |
| **充電切れ・再起動後の再開** | Mac再起動後に .command を押して「送信開始」を押せば results.csv を読んで続きから再開 |

### check_alive_all.js の仕様

- CONCURRENCY: 20（同時20件を並列でURL確認）
- TIMEOUT: 10秒（応答なしは NG 判定）
- 出力: `alive.csv`（フォームURLが有効な施設のみ）、`ng.csv`（除外）
- 入力: SHODANフォーマットのCSVのみ対応（`お問い合わせフォーム` 列が必要）

## results.csv の仕組みと重複防止

```
campaigns/01_kaigo/results.csv
```

- 送信実行のたびに1行ずつ追記される
- ヘッダー行は常に維持される（2件目以降も正常）
- 次回実行時にこのファイルを読んで `フォームURL` が一致するものをスキップ
- **Mac充電が切れた場合**: 再起動後に「送信開始」を押すだけで未送信分から再開
- **処理履歴が不明な場合**: `screenshots/` フォルダの `ERR_XXXX_施設名.png` がエラー記録として残る

## 新しいジャンルを追加する

```bash
mkdir -p campaigns/03_<ジャンル名>
```

`campaigns/03_<ジャンル名>/message.txt` を作成（件名+本文）。  
SHODANのCSVをURL確認にかけて `alive.csv` を生成すれば送信可能になる。

## 注意事項

- **送信は取り消し不可**。必ずドライランで確認してから本番送信する
- `alive.csv` / `results.csv` / `ng.csv` はGit管理対象外（ローカル保存のみ）
- 並列数を上げすぎると同一IPがBot判定されてフォームが弾かれる可能性がある
- 1日3000件を超える場合は日をまたいで分割送信を推奨
- 時間帯制限（09:00〜18:00）は営業時間外の送信を自動でブロックする

## 既知の問題・今後の検討事項

> 最終更新: 2026-05-01

### ⚠️ 「送信完了」の定義問題（未解決）

現在の `send_forms.js` における「送信完了」の判定は以下の通り：

```javascript
await submitBtn.click();
await page.waitForLoadState('networkidle');
return { result: '送信完了', filled };  // ← サンクスページの確認なし
```

**問題点**: 「送信ボタンを押してページが遷移した」というだけで「完了」と判定している。実際には：
- バリデーションエラー（入力形式不正など）で弾かれていても「完了」と記録される
- 確認画面で止まっていても「完了」と記録されるケースがある

**改善案（未実装）**: クリック後のページに「ありがとう」「送信しました」「完了」などの文字が含まれているか確認するステップを追加する。
